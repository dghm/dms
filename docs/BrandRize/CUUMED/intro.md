# 網站重建與 CMS 搬遷討論紀錄

## 文件目的

本文件用於整理 CUUMED（cathetermachine.com）網站改版過程中，
關於 **既有網站結構分析、CMS 搬遷範圍、報價邏輯、風險判斷、
以及未來 WordPress 架構規劃** 的完整討論紀錄。

此文件主要用途為：

- 協助釐清專案邊界，避免「搬遷」、「改版」、「設計」混為一談
- 作為內部決策與報價依據，確保後續對客戶說法一致
- 作為正式提案、報價單、合約內容的整理底稿

本文件 **非對客戶版本**，而是用來幫助專案負責人建立清楚判斷與邏輯順序。

---

## 專案背景摘要

CUUMED 現行網站（cathetermachine.com）為第三方廠商所建置，
後台為自家開發 CMS，內容包含：

- 多層產品分類（Featured Products）
- 製造設備／機台頁面（Catheter Machine）
- 合約製造服務頁
- News / Events
- FAQ
- 公司與品質資訊

原廠商以「年度整包服務」方式收費（約 NT$200,000 / 年），
內容混合 **網站維運、SEO、行銷話術與模糊承諾**，
導致客戶實際不清楚自己付費項目與可交付內容。
![原廠商提供服務](其他/1767594937958.jpg)
本次討論核心在於：

- 若將全站內容遷移至新的 CMS（WordPress）
- 實際需要做哪些事情
- 哪些應屬於一次性工程、哪些應為年度服務
- 視覺設計是否應包含在搬遷中（結論：不應）

## 文件結構大綱

1. 專案範圍界定

   - 什麼是「全內容搬遷到新 CMS」
   - 明確不包含的項目（視覺設計、版型）

2. 全內容搬遷的工作項目拆解

   - 內容盤點與結構重組
   - CMS 架構建立
   - 內容搬遷與資料清洗
   - 測試與上線

3. 搬遷專案的合理報價邏輯

   - 為何不能用頁數計價
   - 一次性費用的合理區間
   - 與原廠商年約 20 萬方案的本質差異

4. 視覺設計與版型的角色定位

   - 為何不能包在搬遷中
   - 套用現成樣版 vs 客製視覺設計
   - 各自的適用情境與加價區間

5. 未來 WordPress CMS 架構建議

   - 建議使用的 Post Types
   - Taxonomy 與分類邏輯
   - 對前台導覽與後台操作的影響

6. 專案風險與實務判斷原則
   - 自家 CMS 搬遷的實際風險
   - 如何避免後續被凹與責任不清
   - 專案分 Phase 的必要性
