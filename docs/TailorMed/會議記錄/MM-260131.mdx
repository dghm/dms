### OMS 資料庫工作：

- OMS 核心資料表有：Order > AIRWAYBILL/POD > Tracking > Release Check List of Collection
- 上述 4 表單需要先著手進行 Automation 自動化以減輕事務人員工作的複雜度(Cody:需要先著手規劃進行)
- Order 業務內容以出口(Export)為多，在 Release Check List of Collection 的呈現用可 Filter 進行資料顯示的篩選
- 完成上述後，OMS 資料庫核心表單亦需要進行 Interface 的建置
- 業務用 Interface 的帳號:sales@tailormed-intl.com

### US 資料庫在初期建置的部份，有幾個部份需要先進行：

- 1.CRM 會有少量自行開發的客戶，目前先要做台灣的業務為主
- 2.OMS 資料庫的建置，因複雜相對高，目前可能處理的量約為 20 至 30 票/每月，故前期先人工於 Excel 或是 Word 處理，待後續的 CRM 跟 FIN 資料庫整備階段完成後再來進行。
- 3.以 MVP 方式先進行 CRM 以及 FIN 資料庫及基 Interface 操作介面的建置，預計 2026 年 3 月中測試完成並啟用上線。
- 4.因之後亦有可能在 JP 也會使用同樣的資料庫的功能及架構，故亦需要提供操作文件，以備後續業務開展使用。
