# 一頁決策摘要

## 專案背景與核心需求

依品質手冊規範，企業需定期對供應商進行年度稽核，以確保其品質體系持續符合要求。  
由於供應商多位於海外，實務上以「問卷填寫」方式進行稽核作業。

目前面臨的實際狀況包括：

- 供應商數量約 10–20 間，未來仍有增加可能
- 稽核單位雖不進行逐筆抽查，但期望看到**可統整的數字與結果**
- QA 作業重點在於：
  - 是否能快速掌握稽核重點
  - 是否能清楚交代稽核結果
  - 是否降低人工整理與回溯資料的時間成本

本專案即針對上述情境，提供三種不同層級的數位化方案。

---

## 三種方案的核心差異（由低至高）

### 方案 A ｜基本線上回覆（填寫為主）

- 以線上表單完成供應商稽核問卷填寫
- 回覆內容以 Email 形式送交內部保存
- 著重「讓供應商能順利填寫並回收資料」

**適合情境**：  
供應商數量少、僅需完成形式上之年度稽核要求。

---

### 方案 B ｜稽核資料集中管理（保存與追溯）

- 稽核問卷透過資料庫型表單填寫
- 所有回覆集中保存，具備一致結構
- 可快速查閱歷史年度資料與附件

**適合情境**：  
稽核作業已成常態流程，需降低人工整理與資料遺失風險。

---

### 方案 C ｜稽核決策輔助與報表輸出強化（QA 導向）

- 建立於方案 B 的資料基礎上
- 將稽核回覆轉為 QA 可快速判讀的重點資訊
- 提供：
  - 稽核重點彙整
  - 關鍵題目統計視角
  - 可列印／匯出之供應商稽核摘要

**核心價值不在於收集更多資料，而在於：**

- 更快做判斷
- 更好對內、對外說明
- 更有效使用既有稽核資料

---

## 三方案成果對照（QA 視角）

| 項目               | 方案 A | 方案 B | 方案 C |
| ------------------ | ------ | ------ | ------ |
| 線上填寫           | ✔      | ✔      | ✔      |
| 資料集中保存       | ✖      | ✔      | ✔      |
| 歷史追溯           | ✖      | ✔      | ✔      |
| 快速掌握稽核重點   | ✖      | 部分   | ✔      |
| 統整數字與概況     | ✖      | 部分   | ✔      |
| 稽核摘要列印／匯出 | ✖      | ✖      | ✔      |
| QA 決策輔助        | ✖      | ✖      | ✔      |

---

## 建議導入策略

- **初期或供應商數量較少**：可從方案 A 開始
- **希望降低人工整理與資料風險**：建議至少導入方案 B
- **稽核已成固定年度作業，需提升效率與交代能力**：適合導入方案 C

> 三種方案可視實際需求分階段導入，  
> 不需一次到位，也不影響既有作業流程。

---

## 決策重點提醒

- 本專案不改變 QA 的專業判斷責任
- 所有方案皆以現行稽核流程為基礎進行優化
- 方案 C 著重於「讓資料更好用」，而非增加系統複雜度
