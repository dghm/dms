# 業務 Interface 與資料呈現建置工作說明（SOW 草稿）

本文件說明本階段針對「業務資料呈現、判讀與執行輔助」所進行之工作項目與可交付成果，重點在於**資料結構補強與 Interface 呈現**，不涉及進階自動化或系統整合。

---

## 一、業務人員 Read-Only Interface 建置（立即可用）

### 工作項目

- 建立業務人員專用 Read-Only Interface
- 建置以下三個模組之 List 與 Detail 檢視：
  1. 所有訂單（Quotations）
  2. 客戶列表（Partners）
  3. 聯絡人名單（Contacts / ATTNs）
- 設定清單頁與詳細頁之欄位顯示差異
- 限制使用者僅能檢視資料，不可直接編輯核心欄位

### 可達成成果

- 業務人員可即時查詢訂單、客戶與聯絡人資訊
- 無需進入 Airtable Base 即可完成日常查詢
- 降低誤編輯風險，確保資料一致性
- 作為後續進階功能（Dashboard、編輯權限）的基礎介面

---

## 二、業務判讀用欄位新增與資料補強（既有三大模組）

### 工作項目

- 於 Quotations、Partners、Contacts 三個 Table 中新增業務判讀用欄位（❇️）
- 設計並建立必要之 Rollup 與 Formula 欄位，例如：
  - 訂單數量彙總
  - 最近訂單日期
  - 客戶活躍度指標
  - 聯絡人實際參與訂單指標
- 調整欄位命名與顯示順序，使其可直接用於判讀

### 可達成成果

- 業務可快速判斷客戶與聯絡人的實際商業價值
- 現有資料由「名單」轉為「可解讀資訊」
- 不需額外報表即可於清單層級進行初步決策判斷

---

## 三、Historical Data 表單設計（過往業務資料建置）

### 工作項目

- 新增 Historical Sales / Historical Deals 資料表
- 定義僅限業務規劃必要之欄位結構，包括但不限於：
  - 成交日期（Deal Date）
  - 年份（Year，公式欄位）
  - 季度（Quarter，公式欄位）
  - 營收（Revenue）
  - 成本（Cost）
  - 毛利或毛利率（Gross Profit / Gross Margin）
  - 客戶與業務類型
- 設計適合人工補登歷史資料的填寫結構
- 視需求建立與既有客戶或訂單資料之連動關係

### 可達成成果

- 客戶可將過去 1–2 年完成訂單資料集中建置
- 歷史資料不再分散於 Word 或 Excel
- 建立可供系統分析與彙總的歷史業務資料層
- 作為年度與季度業務規劃的資料基礎

---

## 四、業務規劃用 Dashboard Interface（資料呈現為主）

### 工作項目

- 建立業務分析用 Dashboard Interface（Read-Only）
- 彙總 Historical Data 所產生之關鍵指標，例如：
  - 年度／季度營收概覽
  - 訂單數量趨勢
  - 客戶貢獻度分佈
- 以圖表與摘要數值方式呈現
- 設計可快速判斷下一步行動的資訊排列邏輯

### 可達成成果

- 業務主管可快速掌握各年度與季度的表現差異
- 支援年度與季度業績規劃與檢討
- 決策討論不需再仰賴外部試算表

---

## 五、進行中訂單回報與跟進（限定編輯 Interface）

### 工作項目

- 建立業務人員限定編輯之操作介面
- 僅開放必要欄位可由業務人員填寫，例如：
  - 跟進狀態
  - Follow-up Date
  - 業務備註與回報內容
- 其餘核心資料維持唯讀狀態
- 調整操作流程以符合實務使用情境

### 可達成成果

- 業務人員可於同一介面完成查詢與回報
- 管理層可即時掌握業務執行與跟進狀況
- 在不影響資料安全的前提下提升回報完整度

---

## 六、整體交付成果總結

完成本階段後，系統將具備以下能力：

- 可查詢：業務人員快速取得所需資訊
- 可判讀：資料已轉化為可解釋的業務指標
- 可規劃：具備歷史資料支援年度與季度規劃
- 可回報：業務進行中狀態可被有效回收與追蹤

本結構亦保留後續擴充空間，以利未來導入主管 Dashboard、進階分析或自動化流程。
