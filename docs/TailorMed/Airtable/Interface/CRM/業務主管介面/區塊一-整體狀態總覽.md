# 區塊一｜整體狀態總覽

本區塊是業務主管打開 Dashboard 時**第一個、也是最重要的閱讀區域**。  
其目的並非提供細節，而是協助主管在極短時間內回答一個問題：

> **目前整體報價狀況，是正常，還是已經出現需要注意的訊號？**

---

## 一、為什麼一定要有「整體狀態總覽」

在沒有整體視角的情況下，所有風險與警示都會失去參考基準。

若直接呈現：

- 個別報價清單
- 單一業務人員狀況
- 個案的異常提醒

業務主管容易陷入「局部問題」，而無法判斷其對整體的影響程度。

因此，本 Dashboard 的第一層設計，刻意先提供**全貌判斷**，再引導進入細部。

---

## 二、此區塊回答的核心問題

「整體狀態總覽」僅負責回答以下三個管理問題：

1. 目前系統中有多少報價正在進行中？
2. 這些報價是否仍在合理的推進節奏內？
3. 整體狀況看起來偏向穩定，還是已有失衡跡象？

若這三個問題無法在此區塊被回答，代表 Dashboard 的起點設計失焦。

---

## 三、整體狀態的判斷原則

本區塊的狀態判斷，並非以「單一數值」作為依據，而是透過**狀態分布的結構**來觀察。

例如：

- 大多數報價集中於合理的進行狀態  
  → 可視為正常流動
- 大量報價停留在同一進度，且時間拉長  
  → 潛在停滯風險
- 少數高金額案件卡住  
  → 需提高注意力

重點不在於精準預測結果，而在於**是否出現異常集中現象**。

---

## 四、為什麼不在此區塊顯示細節

「整體狀態總覽」刻意不顯示以下資訊：

- 單筆報價的完整內容
- 詳細的洽談紀錄
- 個別客戶或業務的完整列表

原因在於：

> **第一層畫面若出現過多細節，主管將無法快速形成判斷。**

細節應在後續區塊中，於「已確認需要關注」的前提下再被展開。

---

## 五、此區塊與後續區塊的關係

本區塊的角色是「判斷是否正常」，而非「指出問題來源」。

當整體狀態呈現異常時，業務主管才需要進一步查看：

- 哪些案件停滯
- 風險集中在哪裡
- 是否需要介入或調整資源

這些問題，將由後續的風險與介入區塊接手處理。

---

## 六、小結

「整體狀態總覽」的成功與否，不取決於顯示多少資訊，而取決於：

- 業務主管是否能在第一眼，判斷目前是否需要關注
- 是否能自然引導其進入下一層分析

**當整體狀態出現異常時，Dashboard 如何協助找出停滯與風險來源。**
