## 2. 文件目的

本文件用於記錄 CRM、OMS 與 FIN Airtable 資料庫系統之使用者驗收測試（User Acceptance Test, UAT）內容與結果，確認系統功能符合實際業務操作需求，並作為正式上線前之驗收依據。

## 3. 測試範圍

| 測試模組                 | 英文名稱        | 說明                                           |
| ------------------------ | --------------- | ---------------------------------------------- |
| CRM 資料庫               | CRM Database    | 客戶資料管理、報價單（Quotation）產出與追蹤    |
| OMS 資料庫               | OMS Database    | 訂單管理、航空提單（AWB）及交貨證明（POD）產出 |
| FIN 資料庫               | FIN Database    | 財務管理、借記通知單（Debit Note）產出         |
| 關聯表（Linked Records） | Linked Records  | 三資料庫間多對多關聯關係驗證                   |
| 自動化流程               | Automation      | 觸發條件、通知、欄位更新等自動化功能驗證       |
| 視圖與報表               | Views / Reports | Grid 視圖及篩選、排序功能驗證                  |

## 4. 測試人員與職責

| 姓名   | 角色           | 所屬單位             | 職責說明                                   |
| ------ | -------------- | -------------------- | ------------------------------------------ |
| 謝萬錤 | 開發方代表     | 萬能數維有限公司     | 提供測試環境、說明功能規格、協助問題排查   |
|        | 使用方代表     | 崴宇國際物流有限公司 | 執行測試案例、確認測試結果、簽署驗收確認書 |
|        | 見證人（選填） | 崴宇國際物流有限公司 | 見證驗收過程，確保公正性                   |

## 5. 測試時程

| 日期               | 階段         | 負責方 | 說明                                        |
| ------------------ | ------------ | ------ | ------------------------------------------- |
| 2026/01/17 – 01/23 | 測試環境準備 | 開發方 | Airtable 工作區開通、帳號授權、測試資料填入 |
| 2026/01/24 – 01/28 | 測試案例執行 | 使用方 | 逐項執行測試案例，記錄實際結果              |
| 2026/01/29 – 01/30 | 缺陷修正     | 開發方 | 針對測試失敗項目進行修正                    |
| 2026/01/31         | 驗收簽署     | 雙方   | 確認全部案例通過，簽署驗收確認書            |

## 6. 驗收標準

| 判定結果                       | 說明                                               |
| ------------------------------ | -------------------------------------------------- |
| 通過（Pass）                   | 測試案例執行結果與預期結果完全符合                 |
| 有條件通過（Conditional Pass） | 存在輕微缺陷，雙方同意列為後續改善項目，不影響驗收 |
| 失敗（Fail）                   | 實際結果與預期不符，需開發方修正後重測             |

## 7. 測試填寫說明

- **Actual Result** 應簡要描述實際操作結果，而非僅勾選通過或失敗。
- 若測試結果為 **失敗**，需同步於「缺陷追蹤紀錄」登錄問題。
- 若無缺陷，可於缺陷紀錄區註明「本次驗收無缺陷」。
- 本文件以使用方實際操作結果為主，作為最終驗收依據。
