## 本表用途說明

本權限矩陣用於說明不同使用者角色，在目前 CRM 架構下  
各自負責與可操作的資訊範圍。

本版本 CRM 尚未導入獨立的 Action / Task 機制，  
因此「行動責任」透過案件進度的更新來反映。

---

## 角色定義摘要

| 角色     | 核心責任                               |
| -------- | -------------------------------------- |
| 業務主管 | 報價文件正確性、案件指派、管理介入判斷 |
| 業務人員 | 案件實際進度的真實回報                 |

---

## 一、Quotations（報價單）權限矩陣

| 項目                                                         | 業務主管 | 業務人員 |
| ------------------------------------------------------------ | -------- | -------- |
| 建立報價單                                                   | ✅       | ❌       |
| 編輯報價金額與條件                                           | ✅       | ❌       |
| 編輯文件狀態（Draft / Sent / Accepted / Rejected / Expired） | ✅       | ❌       |
| 指派案件負責業務（Case Owner）                               | ✅       | ❌       |
| 檢視自己負責的報價單                                         | ✅       | ✅       |
| 檢視所有報價單                                               | ✅       | ❌       |
| 更新業務進度（Sales Stage）                                  | ❌       | ✅       |
| 更新最後跟進時間（Last Follow-up）                           | ❌       | ✅       |

---

## 二、Sales Stage（業務進度）責任矩陣

| 項目                     | 業務主管 | 業務人員 |
| ------------------------ | -------- | -------- |
| 定義進度選項（制度設計） | ✅       | ❌       |
| 更新案件進度             | ❌       | ✅       |
| 檢視進度分布             | ✅       | ❌       |
| 依進度與停滯判斷是否介入 | ✅       | ❌       |

> 說明：  
> 在本版本中，Sales Stage 即為案件行動結果的呈現。

---

## 三、停滯與風險判斷責任

| 項目             | 業務主管 | 業務人員 |
| ---------------- | -------- | -------- |
| 判斷案件是否停滯 | ✅       | ❌       |
| 判斷是否需要介入 | ✅       | ❌       |
| 說明實際客戶狀況 | ❌       | ✅       |

---

## 四、為何本階段不導入 Action / Tasks

| 原因       | 說明                       |
| ---------- | -------------------------- |
| 使用成熟度 | 使用者尚未建立穩定回報習慣 |
| 管理成本   | 任務管理會增加初期負擔     |
| 重複功能   | 行動結果已反映於進度變化   |
| 導入風險   | 易被視為監控工具           |

---

## 五、後續可擴充方向（非本階段）

未來若組織成熟度提升，可考慮：

- 將特定進度階段拆出 Action 機制
- 僅限高金額或高風險案件使用
- 不影響既有進度回報流程

---

## 小結

在目前版本中：

> **案件是否被推進，就是行動是否發生的最佳證據。**

透過進度更新與停滯判斷，即可支撐管理決策，  
無需額外的任務系統。
