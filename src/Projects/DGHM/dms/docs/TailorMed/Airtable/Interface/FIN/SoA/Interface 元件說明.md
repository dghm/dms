## 介面目的說明

本介面用於管理 **帳務對帳單（Statement of Account, SoA）** 的完整流程，  
協助財務與業務主管依指定帳期（YYYY-MM）與請款對象，  
系統化彙整可請款項目、產出對帳單，並追蹤已完成之 SoA 紀錄。

整體流程以「**狀態分區**」方式呈現，避免草稿、待處理與已完成帳務混在一起，  
降低遺漏、重複或誤請款的風險。

---

## 區塊一｜ SoA Draft（帳務對帳單草稿）

本區為帳務對帳單（SoA）的 **草稿建立區**。

使用者可在此建立新的 SoA，並設定：

| 說明                   | 對應欄位      |
| ---------------------- | ------------- |
| 對帳帳期               | Billing Month |
| 請款對象               | Billing To    |
| 請款對象之 Debit Notes | Debit Notes   |
| 收款銀行               | Bank Account  |
| SoA PDF                | Attachment    |

**右上功能按鈕：**

- **[Print]**：點選後可另開視窗進入 Page Designer，進行列印或轉出 PDF 檔案

草稿階段的 SoA 尚未寄送，僅作為內部整理與確認用途。  
在此階段，可持續調整納入的 Debit Notes 與相關欄位資訊。

> **使用情境說明**  
> 本區通常由財務或業務主管操作，用於正式請款前的彙整與檢核。

---

## 區塊二｜ Ready for SoA（可納入對帳清單）

本區顯示 **符合條件、可納入帳務對帳單（SoA）** 的 Debit Notes 清單。

系統會依既定規則自動篩選，例如：

| 說明                     | 對應欄位        |
| ------------------------ | --------------- |
| 需納入 SoA 之 Debit Note | SoA?            |
| 尚未請款                 | Received        |
| 是否已納入 SoA 草稿      | Included in SoA |

使用者可從此清單中，選擇要加入 SoA 草稿的項目。

> **設計重點說明**  
> 本清單為系統自動產生，無需人工逐筆查找，  
> 可有效降低漏列或重複請款的風險。

---

## 區塊三｜ SoA Sent（已完成寄送之對帳單）

本區顯示 **已完成編輯並正式寄送** 的帳務對帳單（SoA）紀錄。

使用者可在此：

- 查閱歷史 SoA 清單
- 回溯特定帳期的請款內容
- SoA 中各別 Debit Note 金額及基礎資訊
- 作為後續對帳、收款與稽核依據

**右上功能按鈕：**

- **[Go to SoA Sent]**：點選後可另開視窗進入此 SoA 的 Data View，進行更詳細內容檢視

已寄送之 SoA 原則上不再修改，以確保帳務資料的一致性與可追溯性。

---

## 流程總結

SoA 的處理流程由上而下依序為：

1. **建立 SoA 草稿（Draft）**
2. **篩選並納入符合條件的 Debit Notes**
3. **完成寄送並留存 SoA 紀錄**

透過分區設計，使用者可快速辨識帳務目前所處階段，  
避免不同狀態的資料混用，提升帳務管理效率與透明度。
